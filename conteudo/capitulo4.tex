\chapter{Proposta de Solução}\label{cpProposta}

O presente trabalho tem por objetivo realizar a evolução do gerenciamento do portal de algoritmos. Para tanto, foi realizada a engenharia reversa do \textit{software} atual, através da análise do código fonte, suas funcionalidades, sua arquitetura e seu banco de dados.

Para descrever a evolução de \textit{software}, nas seções seguintes serão apresentados conceitos e artefatos da engenharia de \textit{software}. Utilizando-se de artefatos da metodologia ICONIX será modelado o \textit{software}.

A solução proposta será desenvolvida na linguagem de programação Java, a fim de unificar as tecnologias do gerenciamento do portal de algoritmos com seu analisador algorítmico.

O \textit{software} será construído com uma arquitetura orientada a serviços, cujo objetivo é que ele seja disponibilizado em interfaces, ou seja, seja acessível através de \ac{REST}.

Na evolução do \textit{software} serão mantidas as funcionalidades atuais e adicionadas novas funcionalidas:

\begin{itemize}
	\item Manter Administradores.
	\item Manter Grupos de Administradores.
	\item Manter Usuários, Alunos e Professores.
	\item Manter Grupos de Alunos.
	\item Manter Instituições.
	\item Manter Permissões.
	\item Manter Países, Estados e Cidades.
	\item Manter Tipos de Problemas.
	\item Manter Problemas.
	\item Manter Entradas e Saídas
	\item Manter Palavras-chave
	\item Manter Listas de Problemas
	\item Manter Soluções de Problemas
\end{itemize}

\section{Interfaces Gráficas}\label{scPrototipos}

Os protótipos exibidos nessa seção procuram seguir os seguintes princípios de usabilidade conforme \citet{Benyon2011}:

\begin{enumerate}
	\item Visibilidade: garante que os elementos sejam visíveis, de forma que as pessoas possam ver quais funções estão disponíveis e o que o sistema está fazendo atualmente.
	\item Consistência: mantém consistência no uso de características de \textit{design} e com sistema semelhantes e método-padrão de trabalho.
	\item Familiaridade: utiliza linguagem e símbolos com os quais os futuros usuários estão familiarizados.
	\item \textit{Affordance}: criar \textit{design} de forma que fique claro para o quê elas servem.
	\item Navegação: proporcione suporte para que as pessoas possam se movimentar pelo sistema.
	\item Retorno: retorna as informações rapidamente a informação do sistema para os usuários para que elas saibam que efeito suas ações causaram.
	\item Estilo: \textit{designs} devem ser elegantes e atraentes.
\end{enumerate}

\section{Arquitetura do Software}

Para o desenvolvimento da evolução do portal de algoritmos foi definido utilizar a arquitetura de \textit{software} \ac{MVC}, estas representadas na Figura \ref{imgArquitetura}.

\FloatBarrier
\begin{figure}[!htb]
	\centering
	\caption{Arquitetura Lógica do Portal de Algoritmos}
	\includegraphics[width=15cm]{imagens/arquitetura-logica.png}
	\label{imgArquitetura}
	Fonte: (AUTOR, 2018)
\end{figure}
\FloatBarrier

Na camada de \textit{Model} é feita a manipulação de dados, leitura, escrita de dados.

Na camada de \textit{Controller} é responsável por receber todas as requisições do usuário, nessa camada possui métodos de ações.

Na camada de \textit{View} temos a interação com o usuário, essa camada faz apenas a exibição dos dados.

\newpage

Além da arquitetura lógica, temos a arquitetura de sistema, que é apresentado na Figura \ref{imgArquiteturaSistema}.

\FloatBarrier
\begin{figure}[!htb]
	\centering
	\caption{Arquitetura do Software do Portal de Algoritmos}
	\includegraphics[width=15cm]{imagens/arquitetura-software.png}
	\label{imgArquiteturaSistema}
	Fonte: (AUTOR, 2018)
\end{figure}
\FloatBarrier

Na camada de \textit{View e Controller} utilizamos duas tecnologias, \textit{AngularJS 4 e JAX-RS} com a implementação \textit{Jersey 2.26}, essas tecnologias são responsáveis pela exibição do dados e comunicação com a camada de modelos.

A camada de \textit{POJO e JAVA 8} é responsável pela manipulação dos dados transformando-os em objetos interpretados pelo sistema operacional.

Na camada de persistência temos o \ac{JDBC} que é responsável pela comunicação com o banco de dados \textit{Postgresql} 10.
 
Até o presente momento foi apresentado a modelagem e estrutura do novo \textit{software}, a seguir é sugerido a configuração de segurança para o servidor de aplicação.
